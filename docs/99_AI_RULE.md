# 목적

이 문서는 매출연구소 프로젝트에 참여하는 모든 AI(ChatGPT, Codex 등)가 반드시 따라야 하는 작업 규칙이다.

AI는 항상 프로젝트의 목표를 이해한 상태에서 작업해야 한다.

---
# AI 행동 원칙

AI는 사용자의 요구를 그대로 구현하는 것이 아니라,

프로젝트의 목표를 고려하여 더 나은 방법이 있다면 이유를 설명하고 제안할 수 있다.

단,

사용자의 승인 없이 프로젝트 방향을 변경해서는 안 된다.

# 프로젝트 최우선 규칙

가장 먼저 아래 문서를 읽는다.

docs/00_PROJECT_RULE.md

---

# 프로젝트 목표

매출연구소는 단순한 홈페이지가 아니다.

계약을 만드는 영업 도구이다.

모든 구현은 아래 질문을 기준으로 한다.

> "이 기능이 계약 가능성을 높이는가?"

---

# 개발 철학

- 화려함보다 신뢰감을 우선한다.
- 기능보다 사용자 경험을 우선한다.
- 디자인보다 브랜드를 우선한다.
- 코드는 단순하고 유지보수하기 쉽게 작성한다.

---

# 기술 스택

초기 프로젝트에서는 아래 기술만 사용한다.

- HTML5
- CSS3
- Vanilla JavaScript

사용 금지

- React
- Vue
- Angular
- jQuery
- Bootstrap
- Tailwind CSS

필요성이 생길 때까지 외부 라이브러리는 사용하지 않는다.

---

# 프로젝트 구조

AI는 기존 폴더 구조를 임의로 변경하지 않는다.

예시

maechul-lab/

docs/

assets/

css/

js/

index.html

README.md

---

# 작업 원칙

한 번에 큰 기능을 구현하지 않는다.

항상 작은 단위로 작업한다.

예시

✅ Intro Animation

↓

✅ Hero Section

↓

✅ Sticky Header

↓

✅ Portfolio Filter

↓

✅ Contact

---

# 코드 작성 규칙

- 코드의 가독성을 우선한다.
- 불필요하게 복잡한 코드를 작성하지 않는다.
- 함수는 역할별로 분리한다.
- CSS는 재사용 가능하게 작성한다.
- JavaScript는 모듈화가 가능하도록 작성한다.
- 주석은 필요한 곳에만 작성한다.

---

# 반응형 규칙

모바일을 가장 먼저 고려한다.

지원 범위

- 320px 이상
- 태블릿
- 노트북
- 데스크톱

모든 화면에서 UI가 깨지지 않아야 한다.

---

# 접근성 규칙

- 글자는 읽기 쉬워야 한다.
- 버튼은 충분히 크게 만든다.
- 대비를 유지한다.
- 키보드 탐색을 고려한다.
- 의미 있는 HTML 태그를 사용한다.

---

# 성능 규칙

- 빠른 로딩을 우선한다.
- 최소한의 JavaScript만 사용한다.
- 불필요한 애니메이션을 만들지 않는다.
- 이미지와 영상은 최적화를 고려한다.

---

# 디자인 규칙

애니메이션은 의미가 있을 때만 사용한다.

불필요한 움직임은 만들지 않는다.

브랜드의 신뢰감을 해치는 효과는 사용하지 않는다.

---

# 수정 규칙

AI는 사용자의 요청 없이 아래 내용을 변경하지 않는다.

- 프로젝트 구조
- 디자인 방향
- UX 흐름
- 브랜드 철학

변경이 필요하면 먼저 이유를 설명하고 승인을 요청한다.

---

# 응답 규칙

작업이 끝나면 반드시 아래 내용을 함께 제공한다.

1. 수정한 파일

2. 변경한 내용

3. 변경한 이유

4. 테스트 방법

---

# 금지 사항

AI는 아래 행동을 하지 않는다.

- 임의로 라이브러리 추가
- 임의로 파일 구조 변경
- 임의로 디자인 변경
- 임의로 UX 변경
- 사용하지 않는 코드 추가
- 과도한 애니메이션 추가

---

# 최종 목표

모든 구현은

"예쁜 홈페이지"

가 아니라

"계약이 잘 되는 홈페이지"

를 만드는 것을 목표로 한다.

사용자는

"와, 멋있다."

보다

"이 사람이라면 믿고 맡길 수 있겠다."

라는 생각이 들어야 한다.

매출연구소는

조회수를 만드는 브랜드가 아니라

사장님의 매출을 연구하는 브랜드이다.
