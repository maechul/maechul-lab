\# 프로젝트 정의



매출연구소는 단순한 홈페이지를 만드는 프로젝트가 아니다.



사장님의 문의와 계약을 만드는 브랜드를 구축하는 프로젝트이다.



모든 기획, 디자인, UX, 개발은 이 목표를 기준으로 판단한다.



\---



\# 1. 프로젝트 목표



매출연구소 홈페이지는 회사 소개 사이트가 아니다.



계약을 만드는 영업 도구(Landing Website)이다.



모든 페이지와 기능은 방문자가 자연스럽게 문의를 남기도록 설계한다.



\---



\# 2. 기능 추가 기준



새로운 기능을 추가하기 전에 반드시 아래 질문을 한다.



> \*\*"이 기능이 계약 가능성을 높이는가?"\*\*



\- YES → 추가

\- NO → 만들지 않는다.



멋있어 보이는 기능보다

실제로 계약률을 높이는 기능을 우선한다.



\---



\# 3. 디자인 원칙



\- 화려함보다 신뢰감을 우선한다.

\- 미니멀한 디자인을 지향한다.

\- 애니메이션은 최소화한다.

\- 불필요한 장식은 사용하지 않는다.

\- 여백을 적극 활용한다.

\- 브랜드가 돋보이도록 디자인한다.

\- 40\~60대 사장님도 쉽게 사용할 수 있어야 한다.



\---



\# 4. UX 원칙



\- 클릭 수를 최소화한다.

\- 사용자가 고민하지 않게 만든다.

\- CTA(문의하기)는 항상 쉽게 찾을 수 있어야 한다.

\- 모든 페이지는 문의까지 이어져야 한다.

\- 사용자는 무엇을 해야 할지 항상 명확해야 한다.



\---



\# 5. 접근성(Accessibility)



모든 사용자가 쉽게 이용할 수 있어야 한다.



\- 글자는 읽기 쉬워야 한다.

\- 버튼은 충분히 크게 만든다.

\- 명확한 색 대비를 유지한다.

\- 색상만으로 정보를 전달하지 않는다.

\- 모바일에서 손가락으로 누르기 쉽게 설계한다.



\---



\# 6. Mobile First



모바일을 기준으로 설계한다.



지원 목표



| 기기 | 최소 폭 |

|------|---------|

| 작은 스마트폰 | 320px |

| 일반 스마트폰 | 360\~430px |

| 태블릿 | 768px |

| 노트북 | 1280px |

| 데스크톱 | 1440px 이상 |



지원 대상



\- 갤럭시 A 시리즈

\- 갤럭시 S9 이상

\- 아이폰 SE 이상

\- 보급형 안드로이드

\- 태블릿

\- 노트북

\- 데스크톱

모든 해상도에서
UI가 깨지지 않아야 한다.

어느 한 화면을 기준으로 제작하지 않는다.

모든 기기에서 자연스럽게 동작하도록 반응형을 구현한다.

\---



\# 7. 브라우저 지원



아래 브라우저에서 정상 동작해야 한다.



\- Chrome

\- Samsung Internet

\- Safari

\- Edge

\- Android Chrome



최신 버전뿐 아니라 최근 5년 정도의 브라우저에서도 최대한 동일한 경험을 제공한다.



\---



\# 8. 성능 목표



홈페이지는 빠르고 가벼워야 한다.



\- 빠른 초기 로딩

\- 이미지 최적화(PNG / JPG / JPEG 우선)

\- Lazy Loading 사용

\- 불필요한 라이브러리 사용 금지

\- 최소한의 JavaScript 사용

\- 애니메이션 최소화



사용자가 3초 이상 기다리게 만들지 않는다.



\---



\# 9. 개발 원칙



초기 버전 기술 스택



\- HTML5

\- CSS3

\- Vanilla JavaScript



초기 버전에서는



\- React 사용하지 않는다.

\- Vue 사용하지 않는다.

\- 정적인 사이트로 제작한다.



확장 가능하도록 구조만 설계한다.



\---



\# 10. 코드 원칙



모든 코드는 사람이 읽기 쉬워야 한다.



\- 역할별 파일 분리

\- 중복 코드 최소화

\- 충분한 주석 작성

\- 유지보수가 쉬운 구조

\- 일관된 네이밍 규칙 사용



\---



\# 11. 브랜드 원칙



매출연구소는



영상 제작 업체처럼 보이지 않는다.



브랜드처럼 보여야 한다.



전문가처럼 보여야 한다.



연구하는 회사처럼 보여야 한다.



조회수보다



사장님의 매출을 연구하는 브랜드여야 한다.



\---



\# 12. 개발 프로세스



절대로 바로 개발하지 않는다.



항상 아래 순서를 따른다.



1\. 기획

2\. IA

3\. User Flow

4\. WireFrame

5\. UI 설계

6\. UX 검토

7\. 디자인 시스템

8\. 개발

9\. 테스트

10\. 배포



\---



\# 13. 품질 기준



\- 모바일에서 먼저 테스트한다.

\- 새로운 기능은 기존 기능을 망가뜨리지 않아야 한다.

\- 작은 단위로 커밋한다.

\- 항상 이전 버전으로 복구 가능해야 한다.

\- 사용자 경험을 코드보다 우선한다.



\---



\# 14. 프로젝트 의사결정 원칙



의견이 충돌할 경우 아래 순서를 따른다.



1\. 브랜드 신뢰

2\. 계약 전환율

3\. UX

4\. 디자인

5\. 개발 편의성



항상 브랜드 신뢰를 가장 우선한다.



\---



\# 최종 목표



사용자가



> "와, 예쁜 홈페이지다."



라고 말하는 것이 목표가 아니다.



사용자가



> "이 사람이라면 믿고 맡길 수 있겠다."



라고 생각하게 만드는 것이 목표이다.



매출연구소 홈페이지는



\*\*브랜드를 만드는 홈페이지가 아니라 계약을 만드는 홈페이지\*\*이다.
